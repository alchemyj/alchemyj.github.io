    Setting up the license file

    An AlchemyJ license key is required to run an AlchemyJ generated API. You should put the license file on the server which deployed the AlchemyJ project.

    There are 3 ways you can choose to set up the license file.

    License Key File Environment Variable.

    To create an environment on the operating system, its value is the file path of the license file. To specify the variable, you can go to the AlchemyJ toolbar, click the Properties and select the "License" tab on the popup window.

    External configuration.

    1. If it is a Java API project, the license key location should be provided when initializing the ContextHolder object. The key name is "alchemyj.license-file-location". For more details, please refer to the Setting Application Properties.

    2. For the REST projects, you can define the path of the application configuration yml file in REST API worksheet. In the yml file, change the key value of "alchemyj/license-file-location" to the updated license key file path.

    License key file location.

    To specify the path of the license file directly. You can specify the file path on the Properties popup window.

    Priority

    If the license is specified in multiple ways, Environment Variable will be the top priority. The external configuration is the second while license key file location is the last priority.

    • External configuration.
    • License key file location.
