    Define your own testing data

    The random data generation program only generates one test case JSON file for each function point. It can be a sample file to follow when adding more test cases are needed. Each JSON file contains one set of test data and the expected result for one test case.

    For the unit test generated by the compiler for Java API, there would be only one JSON file for each function point. You may create more JSON files under the same folder if you need more test cases. The file name must start with FunctionPointName--. The unit test program will pick up these files and run them according to the function name prefix in the file name.

    Following is the content of the above JSON file.

    {
  "methodParameter" : {
    "name" : "BAMMgVx"
  },
  "__expectResult" : {
    "__result" : { }
  }
}

    To define the test value of input parameters, modify the nodes under methodParameter. The node name contains the input parameter name. The value is the values to be inputted.

    You may also define an expected result for a case. To do that, remove the "--" from "__ExpectedResult" and "__Result" and set the expected value into the result node. If the output of the function equals the value in the result node, the test result is passed. If the ExpectedResult node is not defined, a test case is considered passed if it runs with no error.

    The example below shows that there is one input "price" of the function and the input value is 100. The expected value is 1000. If the return is 1000, the test result should be passed. Otherwise, it should be failed.

    {
  "methodParameter" : {
    "price" : 100
  },
  "expectResult" : {
    "result" : 1000
  }
}
