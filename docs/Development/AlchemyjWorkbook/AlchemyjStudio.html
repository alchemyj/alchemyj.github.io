AlchemyJ Studio

AlchemyJ Studio turns Excel into an API modelling environment. It is an Excel Add-in which adds a AlchemyJ menu item in Excel toolbar. You can learn how to add AlchemyJ Studio in Excel in Set up your Microsoft Excel.

AlchemyJ Studio tab provides many tools. This guide provides a description of how to use these tools.

New Project

Turn the current workbook into an AlchemyJ workbook but adding the following required system worksheets:

  • System Run-time Worksheet controls the execution of the program. DO NOT change anything on this worksheet.

  • System Configuration Worksheet contains AlchemyJ Studio configurations and some compiler options.

  • Application Config Worksheet contains application configuration for the API.

Beside adding the above worksheets, New Project also adds a VBA module to the workbook for supporting AlchemyJ Excel Extended Functions. This, however, requires special permission in Excel. Following the below steps to enable this permission.

  1. Go to File > Options > Trust Center.

  2. Click Trust Center Settings.

  3. Under Macro Settings, make sure Trust access to the VBA project object model is checked.

If this permission is not enabled, you will see the following error when clicking New Project.

In case that this permission cannot be enabled (e.g, due to security concern), you may still use AlchemyJ but the New Project button will not work. You can create a New AlchemyJ Workbook by making a copy of Blank AlchemyJ workbook.xlsm found in Sample Models folder in AlchemyJ installation package.

New Definition

Adds a new definition worksheet to the workbook.

Java Class

Adds a Java Class definition worksheet. One Java API may have multiple Java class definition worksheets. The sheet name can be renamed as needed.

Java Package

Adds a Java Package Definition Worksheet which defines a Java package. A Java API must have one and only one Java Package Definition Worksheet. The sheet name can be renamed as needed.

REST Endpoint

Adds a REST Endpoint Worksheet. One REST API may have multiple Endpoint worksheets. The sheet name can be renamed as needed.

REST Controller

Adds a REST Controller Worksheet which defines a RESET API. Each REST API must have one and only one Java Package Definition Worksheet. The sheet name can be renamed as needed.

REST Security Authentication

Adds a Security Authentication Worksheet. It defines the authentication feature of a REST API. One REST API may have at most two Security Authentication Worksheets (One for username password token authentication. One for JWT Token Authentication). This worksheet is only needed if your API requires access control. The sheet name can be renamed as needed.

REST Security User Details Service

Adds a Security User Details Service Worksheet. It defines how user information of a logged in user should be obtained in a REST API. Only one Security User Details Service Worksheet can be added. This worksheet is only needed if your API requires user information lookup. The sheet name can be renamed as needed.

REST Security Resource Auth

Adds a Security Resource Auth Worksheet. It defines endpoint access control. Only one Security Resource Auth Worksheet can be added. This worksheet is only needed if your REST API endpoint requires access control. The sheet name can be renamed as needed.

New Component

Adds a new Component definition worksheet to the workbook.

Data Schema

Adds a Data Schema Worksheet to the workbook. If the current worksheet is a Data Schema Worksheet, it adds a new table schema into the current Data schema worksheet.

Data Dictionary

Adds a DataDictionary Worksheet to the workbook. One workbook may have multiple Data Dictionary worksheet. The sheet name can be renamed as needed.

Launch Debug Console

Launches the Debug Console.

Calculate Now

Calculates all worksheets of all opened workbooks without running any AlchemyJ function. It is the same as the Calculate now (F9) function in Formulas tab.

Calculate sheet

Calculates the current worksheet without running any AlchemyJ function. It is the same as the Calculate Sheet (Shift+F9) function in Formulas tab.

Run Function Point

Runs an AlchemyJ Function Point. The list of function points to run is retrieved from the Function section of all class Definition Worksheets and Endpoint sections of all REST Endpoint Definition Worksheets in the workbook.

Validate

Validates the setting and formula of a Package. A validation worksheet will be shown if any issue is found. You may click on the Related Cell Address link to go to the worksheet and cell that triggers the error.

Build Project

Builds a project with options. Validation will be triggered before building.

AlchemyJ installation directory - It specifies the location of AlchemyJ Compiler. By default, it looks up the AlchemyJ Compiler\bin folder in the installation folder. If AlchemyJ Compiler is located at a different location, change the path to the bin folder of the AlchemyJ Compiler folder.

Build output directory - It specifies where the generated project should be placed at.

Target project sheet - Select the package you want to build.

Source Type - For REST APIs, you can select Jar or War. Select Jar if you want to test the REST API immediately using the built-in Tomcat server. Select War if you want to deploy the API to an Java application server.

For Java APIs, the only option is Jar.

Compile project into a package - Check this option to generate a package when building the project. Otherwise, only the source code would be generated.

Copy package file to directory - Check this option to make a copy of the package to specified directory.

Open build output directory when finished - Check this option to open the build directory after the project is generated successfully.

Start server when finished - Check this option to start the API with a built-in Tomcat server after the API is compiled successfully. This options is only available for REST API and the Source Type must be Jar.

Build and Run on Server

Build a REST API and start the web server. Validation will be triggered first.
