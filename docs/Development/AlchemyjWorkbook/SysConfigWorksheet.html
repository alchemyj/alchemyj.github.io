System Configuration Worksheet

System Configuration worksheet is a system worksheet that must exist in an AlchemyJ Workbook. It is automatically added when you click New Project on AlchemyJ tab. It contains properties that are used in by AlchemyJ Studio in a name-value pair format. Some of the properties may be modified to customize the behavior of AlchemyJ Studio. The worksheet name is %%SysConfig and it must not be changed.

AlchemyJ Code Generator Local Path

The saved value of AlchemyJ Installation Directory on the Build Project screen. There is no need to change this normally.

Build Output Folder

The saved value of Build Output Directory on the Build Project screen. There is no need to change this normally.

Build Target Sheet

The saved value of Target Project Sheet on the Build Project screen. There is no need to change this normally.

Build Type

The saved value of Build Type on the Build Project screen. There is no need to change this normally.

Build Jar Output Folder

The saved value of Build Option\Source Type option on the Build Project screen. There is no need to change this normally.

Compile Project Into a Package

The saved value of Compile Project into a Package checkbox on the Build Project screen. There is no need to change this normally.

Copy Package File to Folder

The saved value of Copy Package File to Directory option on the Build Project screen. There is no need to change this normally.

DB Connection String

The database connection string for running database related AlchemyJ Excel Extended Functions in AlchemyJ Studio. It is required if the API uses database related AlchemyJ Excel Extended Functions such as ExecuteSql and ExecuteSp.

For Oracle: Provider=OraOLEDB.Oracle.1;Password=[password];Persist Security Info=True;User ID=[user id];Data Source=[data source];Extended Properties=""

For example: Provider=OraOLEDB.Oracle.1;Password=myPassword;Persist Security Info=True;User ID=myUsername;Data Source=192.168.0.1:1521/service_name;Extended Properties=""

For MySQL: Driver={MySQL ODBC 5.2 Unicode Driver}; Server=[server address];DB=[database name];UId=[user id]; PWD=[password];OPTION=3;

Example: Driver={MySQL ODBC 5.2 Unicode Driver}; Server=192.168.0.1;DB=test_db;UId=myUsername; PWD=myPassword;OPTION=3;

Database Type

The database type for DB operation in the API. Acceptable values are Oracle and MySQL. Change it according to your need.

Debug Console Port

It specifies the port that Debug Console will monitor. Change it according to your need.

Debug Mode

It controls whether the project should be built using Debug Mode. Change it according to your need.

  • True - Build the project using debug mode.
  • False - Do not build the project in debug mode.

Debugger Command Action File Location

It specifies the debugger command action file location that Debug Console should look for.

Enable AJ Library Function Check

It controls whether AlchemyJ Studio checks for unsupported Excel Functions in the workbook during validation. Change it according to your need.

  • True - Run the check.
  • False - Do not check.

Enable High Performance Mode Check

It controls whether AlchemyJ Studio should check whether the workbook can be built using High Performance Mode during Build Project. Change this field according to your need.

  • True - Run the check.
  • False - Do not check.

Force turn off High Performance Mode

It controls whether High Performance should be turned off. Change it according to your need.

  • True - Disable high performance mode.
  • False - Enable high performance mode (if the workbook can be built using high performance).

High Performance Mode

It states whether the workbook can be build in high performance mode. It is set by AlchemyJ Studio after it analyses the workbook (When Enable High Performance Mode Check is True).

License File Location

It specifies the location of the license file that the generated project should use.

New Sheet for each Validation Run

It controls whether a new validation worksheet is created on every validation check. Change it according to your need.

  • True - Create a new worksheet for every validation run. The sheet name is Validation-[datetime number].
  • False - Use the same worksheet every time. Old validation results will be removed. The sheet name is Validation.

Open Output Folder

The saved value of Open Build Output Directory When Finished checkbox on the Build Project screen. There is no need to change this normally.

Parameter Separator

The character(s) used to separate multiple parameters in parameter strings of AlchemyJ Excel Extended Functions. The default value is ##. There is no need to change this unless you expect the parameter value may contain ##.

Below is an example of using parameter separator in the parameter argument of Executesql.

RECORD_ID1::21::NUMBER::IN##RECORD_ID2::23::NUMBER::IN##OUTPUT::::NUMBER::OUT

This property has the Excel name AlchemyJ_Parameter_Seperator. It is recommended to use this name to refer to this separator in your formulas.

Parameter Type Separator

The character(s) used to separate multiple parameter properties of a parameter in parameter string of AlchemyJ Excel Extended Functions.

This property as the Excel name AlchemyJ_Parameter_Type_Seperator. It is recommended to use this name to refer to this separator in your formulas.

Start Server When Finished

The saved value of Start Server When Finished checkbox on the Build Project screen. There is no need to change this normally.

Studio Log Level

It controls the log level in AlchemyJ Studio. Change it according to your need.

  • 1 - Debug
  • 2 - Information
  • 3 - Warning
  • 4 - Error

The log files are located at the same folder as the workbook.

Run Validation before Build Project

It controls whether AlchemyJ Studio validates the workbook before building the project. Change it according to your need.

  • True - Run validation before building the project
  • False - Skip validation when building the project

The recommended setting is True as it detects problems in the workbook. However, validation might take a long time to run for large workbooks.

Validation Turn Off Warning Message

It controls whether warning message should be shown in a validation result report. Change it according to your need.

  • True - Warning messages will be included.
  • False - Warning messages will not appear. If there is no error, a validation report will not be created.
