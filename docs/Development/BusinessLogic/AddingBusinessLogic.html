Adding business logic

Any worksheets you currently working on that contains business logic is called business logic sheet(s). It needs to define the Input and output parameters in the Class Definition Worksheet or Rest Endpoint Worksheet.

Here introduce a simple business logic definition about authentication with LDAP.

  1. Usually the LDAP server information and Domain will not be the same for different environments, So define the LDAP Server name and Domain in %%AppConfig

  1. Define a simple Business Worksheet and use the function =AuthenLdap(), in C9, key in following formula. Then the input parameters are user name, password from cell C3 and C4. And the output result is C9 where it returns the authentication result.
=AuthenLdap('%%AppConfig'!B2, '%%AppConfig'!B3, C3, C4,FALSE)

  1. The input parameters are single value in text format, thus set the Parameter Type as Basic data and Parameter Class as String. The input addresses are worksheetname!C3 and worksheetname!C4. The input configurations of this function as below screen shots.

  1. The output parameter is a boolean value, so set the Parameter Type as Basic data and Parameter Class as Boolean. The output address is worksheetname!C9. The output configurations of this function as below screen shots.

  1. Input the Business Logic Worksheet name in the Sheets Details section of Class Definition Sheet to create the linkage between Class Definition Sheet and Business Logic Sheet.
