Class Definition Worksheet

Introduction

A Class Definition Worksheet describes how AlchemyJ Compiler should generate a class. It defines the name of the class, the package it is in, the functions it provides, and the work sheets it uses.

It contains the following sections:

Following sections are provided in the worksheet:

  • Basic Settings

  • Functions

  • Functions input parameters

  • Sheet Details

Basic Settings

Define the basic information for a Java Class.

Item**Description
Configuration TypeKeep it as Java Class. Do NOT change it.
Package NameInput the sub-package Name this class should be in. The actual package this class will be in is [package name].[Sub Package Name]. Class that shall be put into the same package shall have the same package name here.
Class NameDefine the name of this Java Class. It must begin with an alphabet and contain no space and dot.

Functions

It defines a list of functions this class provide and their properties.

ItemDescriptionRemark
Function PointThe name of this function used in AlchemyJ Studio. It can be a friendly name with spaces.The function point list will be shown in AlchemyJ > Run Function Point.
Method NameName of the function in generated Java code. That is, the method name other Java developer will use when calling this function in their project.It must begin with an alphabet and contain no space and dot.
Return TypeThe type of class this function returns.The value can be:

Basic Data – The Return type stores an integer, double, string, date, or Boolean.

Basic Data List – This property stores a list of basic data type defined in property class.

Object – This property is an object. Such object should exist in Object Definition.

Object List – This property stores a list of objects. Such object should exist in Object Definition.
Return ClassThe type of the return value can hold.The value can be:
Integer: A whole number between -2,147,483,648 and 2,147,483,647.

Double: A whole number between 4.9406564584124654 x 10^-324. to 1.7976931348623157 x 10^308
Boolean: TRUE or FALSE

Date: Date value

String: Text value

If Return Type is object or object, the object name should be defined here.
Return Value Cell AddressThe cell address where AlchemyJ should refer to for the output of this function.It should include the sheet name. For example, Sheet1!A2.
When Return Type is Object or Object List, the value of the reference cell should contain a JSON representation of the object.
Return Value Row SpanWhen Return Class is String, this property specifies how many cells are read starting from Return Value Cell Address to the bottom to form the return value.This parameter only applies to string. When a string value is longer than the Excel cell maximum number of character (32727) limitation, AlchemyJ can get the value from multiple consecutive cells. If this feature is not need, this field should be blank.
Return Value Col SpanWhen Return Class is String, this property specifies how many cells are read starting from Return Value Cell Address to the right to form the return value.This parameter only applies to string. When a string value is longer than the Excel cell maximum number of character (32727) limitation, AlchemyJ can get the value from multiple consecutive cells. If this feature is not need, this field should be blank.
List Row SizeThe number of rows the list contains.Only effective when Return Type is an Object List or Basic Data List. If the list should be a column list, this field should also be empty.
List Col SizeThen number of columns the list contains.Only effective when Return type is an Object list or Basic Data List. If the list should be a row list, this field should also be empty.
DB Operation TypeIt defines whether DB connection and transaction management are required.

Blank - This function does not involve any database operations.

Non Transaction – This function has database operation but it does not need Transaction management.

Transaction - This function has database operations. If there is any error, the transaction will be rolled back.

Functions input parameters

Input parameters for defined functions if any

ItemDescriptionRemark
Method NameThe method this input parameter refers to.The value must exist in the Function Definition table Method Name column.
Parameter TypeThe type of class of this parameter.The value can be:

Basic Data – The Return type stores an integer, double, string, date, or Boolean.

Basic Data List – This property stores a list of basic data type defined in property class.

Object – This property is an object. Such object should exist in Object Definition.

Object List – This property stores a list of objects. Such object should exist in Object Definition.
Parameter ClassThe value this parameter may hold.The value can be:
Integer: A whole number between -2,147,483,648 and 2,147,483,647.

Double: A whole number between 4.9406564584124654 x 10^-324. to 1.7976931348623157 x 10^308
Boolean: TRUE or FALSE

Date: Date value

String: Text value

If Return Type is object or object, the object name should be defined here.
Parameter NameName of this parameter.It must begin with an alphabet and contain no space and dot.
Write to Cell AddressThe cell address where this parameter value should be written to.It should include the sheet name. For example, Sheet1!A2.
Object Extraction JsonPathSpecifies the JsonPath that can extract the value of this parameter.AlchemyJ passes in object as a JSON string. The value should be in JsonPath format. For example $.name
Extraction Value TypeThe data type of the extracted field in the object.It can be String, Integer, Double, Date, Boolean.
Extraction Value Date FormatDate format when Extraction Value Type is DateExample: yyyy-mm-dd
Value Row SpanWhen Return Class is String, this property specifies how many cells are read starting from Write to Cell Address to the bottom to form the return value.This parameter only applies to string. When a string value is longer than the Excel cell maximum number of character (32727) limitation, AlchemyJ can get the value from multiple consecutive cells. If this feature is not need, this field should be blank.
Value Col SpanWhen Return Class is String, this property specifies how many cells are read starting from Write to Cell Address to the right to form the return value.This parameter only applies to string. When a string value is longer than the Excel cell maximum number of character (32727) limitation, AlchemyJ can get the value from multiple consecutive cells. If this feature is not need, this field should be blank.
List Row SizeThe number of rows the list contains.Only effective when Parameter Type is an Object List or Basic Data List. If the list should be a column list, this field should also be empty.
List Col SizeThen number of columns the list contains.Only effective when Parameter type is an Object list or Basic Data List. If the list should be a row list, this field should also be empty.

Sheet Details

The Sheet Detail section defines the worksheet and the area this class uses. When the API is generated in High Performance mode, AlchemyJ will automatically detect this information. However, when High Performance is force turned off or when Debug Mode is on, the related sheets and area must be defined in this section manually. It should always contain the three system worksheets: %%AppConfig, %%SysConfig, and %%SysRuntime

ItemDescriptionRemark
Sheet NameName of used sheet for this ClassFor better performance, only include the sheets that are related to this class.
Start AddressThis defines the cell address of the start (top left) of the defined range.For example, A1
End AddressThis defines the cell address of the end (bottom right) of the defined range.Use formula GetLastUsedCellAddress([@[Sheet Name]]) to get the last used cell address.
