Package Definition Worksheet

Introduction

A Package Definition Worksheet describes how AlchemyJ Compiler should generate a package. It defines the name of the package, the classes it contains, and properties for the unit test project.

It contains the following sections:

  • Basic Settings

  • Classes

  • Object Definition

  • Unit Test Settings

Basic Settings

Define the basic information of a Java Package.

ItemDescription
Configuration TypeKeep it as Java Package. Do NOT change it.
Package NameDefine the name of this Java Package. For example com.company.myfirstjavaproject.
Jar nameIf you want to generate a Jar file (Source Type is Jar). Define the it here.
Maven Group IDDefine a Group ID for this package. It should follow Java's package name rules. You can use the same value as package name.
Maven Artifact IDDefine a unique Artifact ID for this project. The Group ID and Artifact ID together should uniquely identify this project across all projects.
Maven VersionVersion number of this Java Project. User of this API may use version number to specify which version of the API to import from a Maven Repository.

Classes

It defines the Class Definition Sheet(s) name included in this Java Package.

Object Definition

Object Definition defines the details of an object. Each row defines one property of an object. For example, the Customer object has 5 properties, so it should have 5 rows in this section.

ItemDescriptionRemark
Object NameName of the objectIt must begin with an alphabet and contain no space and dot. It must not be the same as any Class Name.
Property NameProperty Name of the ObjectIt must start with an alphabet and does not contain space and dot.
Property TypeProperty Type

Basic Data –This property stores an integer, double, string, date, or Boolean.

Basic Data List – This property stores a list of basic data type defined in property class.

Object – This property is an object. Such object should exist in Object Definition.

Object List – This property stores a list of objects. Such object should exist in Object Definition.
Property ClassThe type of value this property may hold

Integer - A whole number between -2,147,483,648 and 2,147,483,647.

Double - A whole number between 4.9406564584124654 x 10^-324. to 1.7976931348623157 x 10^308

Boolean - TRUE or FALSE

Date - Date value and please define the Date Format in next column

String - Text value

If Property Type is object or object, the object name should be defined here.
Date FormatDate FormatIndicate the date format string when Property Class is Date. Otherwise, it should be empty.

Unit Test Settings

This section defines the unit test database information and external resource path.

Refer to Test your API by running unit test for test steps.

Properties Description
unit-test.datasource.driver-class-name

The database driver used in the unit test project. It can be one of the following value:

Oracle: oracle.jdbc.driver.OracleDriver

Mysql: com.mysql.jdbc.Driver
unit-test.datasource.jdbc-url

JDBC URL for the JDBC connection used in the unit test project. Below is the supported format:

Oracle: jdbc:oracle:thin:@[Host]:[Port]:[SID]

Mysql: jdbc:mysql:thin:@[Host]:[Port]:[SID]
unit-test.datasource.username User name of the database connection
unit-test.datasource.password Password of database user
unit-test.resource.path Path of unit test resource files. This folder path specifies where unit test data files should be loaded.
