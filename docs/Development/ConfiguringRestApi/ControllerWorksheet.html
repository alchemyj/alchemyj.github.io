Controller Worksheet

Introduction

A Controller (REST Package Definition Worksheet) describes how AlchemyJ Compiler should generate a REST API package. It defines properties of the API and the worksheets that it should refer to for additional configuration.

It contains the following sections:

  • Basic Settings

  • Classes

  • Runtime Configuration

  • Build-and-Run Web Server Configuration

  • Data Source Settings

  • Session Management Settings

  • Web Security Settings

  • LDAP Settings

  • Microservice Configuration

Basic Settings

Basic settings of the REST Controller.

ItemDescription
Configuration TypeKeep it as ‘REST Project’. Do NOT change it.
Jar/War File NameThe name of the Jar file or the War file.
Base Package NameDefine the name of this Java Package. For example com.company.myfirstjavaproject.
Use Microservices ArchitectureSet it to True if the API should be generated as a microservice. In that case, Eureka server address in Microservice Configuration should be provided. Otherwise the section can be left blank. Note that this is not a production-ready feature.
Maven Group IDDefine a Group ID for this package. It should follow Java's package name rules. You can use the same value as package name.
Maven Artifact IDDefine a unique Artifact ID for this project. The Group ID and Artifact ID together should uniquely identify this project across all projects.
Maven VersionVersion number of this Java Project. User of this API may use version number to specify which version of the API to import from a Maven Repository.
Application Configuration File LocationThis defines where the AlchemyJ API should look for the application configuration file (yml). This property will be ignored if ‘Application Configuration File Environment Variable’ is set.
Application Configuration File Environment VariableThis defines the environment variable name the generated Alchemy API will use to look for the application configuration file path. Using this property is more flexible as the path can be changed after the API has been generated by modifying the environment variable.
Log Configuration File LocationThis defines where the AlchemyJ API should look for the log configuration file (yml). This property will be ignored if ‘Application Configuration File Environment Variable’ is set.
Log Configuration File Location Environment VariableThis defines the environment variable name the generated Alchemy API will use to look for the log configuration file path. Using this property is more flexible as the path can be changed after the API has been generated by modifying the environment variable.

Classes

It defines the Class Definition Sheet(s) name included in this RESTS API Package.

Runtime Configuration

PropertiesDescription
alchemyj.stopOnFormulaError

True - No more formula should run when an AlchemyJ Excel Standard Function returns an error.

False - Subsequence formulas will keep on running even though an error has occurred.
alchemyj.stopOnUserRaiseError

True - No more AlchemyJ Excel Extended Functions should run when an error is raised to the Model Error List.

False - Subsequence AlchemyJ Excel Extended Functions will keep on running even though an error has occurred.
alchemyj.throwOnFormulaErrorTrue - return errors on the System Error List as the API response and return Status Code 502.

False - errors on the System Error List will not trigger an error response. Status Code will still be 200.
alchemyj.throwOnUserRaiseError

True - return errors on the Model Error List as the API response and return Status Code 502.

False - errors on the Model Error List will not trigger an error response. Status Code will still be 200.
alchemyj.generateDebugExcel

True - Generate an Excel file which represents the workbook in memory. It is like a snapshot of the model after a method is run. It is only effective when Debug Mode is on.

False - Do not generate the Excel file.
alchemyj.debugExcelLogPathDefine the file path of the debug Excel file when alchemyj.generateDebugExcel is True
alchemyj.logPathThe log file path.
alchemyj.logLevelOptions are ERROR, WARN, INFO, DEBUG, TRACE
spring.jackson.time-zoneTime zone used when formatting dates. For instance, "America/Los_Angeles" or "GMT+8"
spring.application.nameApplication name

Build-and-Run Web Server Configuration

These are properties used by the embedded web application server in the Jar when it is run.

PropertiesDescription
server.portServer port. The default is 8080.
server.servlet.context-pathDefault /, Optional. Used to construct the URL. If value is /stockorder, the url will be localhost:port/stockorder/endpoint id. Otherwise the URL will be localhost:port/endpoint id,
server.connection-timeoutThe number of milliseconds the server will wait, after accepting a connection, for the request uri line to be presented. The default it 3000.

Data Source Settings

Data source setting for this REST controller, multiple data source settings are supported.

PropertiesDescription
spring.datasource.primary.is-use-jndi

True - Retrieve datasource from JNDI.

False - Use the data source information defined by jdbc-url.

Leave it blank if no data source is needed in this API.
spring.datasource.primary.jndi-nameJNDI Name if JNDI is used.
spring.datasource.primary.driver-class-name

JDBC Driver Name,

Oracle:
oracle.jdbc.driver.OracleDriver

Mysql:
com.mysql.jdbc.Driver

spring.datasource.primary.jdbc-url

JDBC URL

Oracle:
jdbc:oracle:thin:@[Host]:[Port]:[SID]
For example: jdbc:oracle:thin:@192.168.0.12:1521:testdb1

MySQL:
jdbc:mysql://[Host]:[Port]/[DB Name]
For example: jdbc:oracle:thin:@192.168.0.12:1521:testdb1

spring.datasource.primary.usernameUser name for MySql database.
spring.datasource.primary.passwordPassword for the user name
spring.datasource.primary.typeKeep it as com.zaxxer.hikari.HikariDataSource. Do not change it.
spring.datasource.primary.minimum-idleThe minimum number of idle connection. The default is default 5.
spring.datasource.primary.maximun-pool-sizeThe maximum number of connections allowed in the pool. The default is 15.
spring.datasource.primary.connection-timeoutConnection Timeout in milliseconds. The default is 30000ms.
spring.datasource.primary.connection-test-queryThe SQL used to test the DB connection. Use SELECT 1 FROM DUAL if you do not have a specific test query. To disable datasource connection test, leave it empty.
spring.datasource.primary.idle-timeoutIdle Timeout in milliseconds. The default is 30000ms

Session Management Settings

PropertiesDescription
spring.session.store-type

Where session information should be stored.

JDBC – Stored the Session information in database.

Server Native – Store the session information on the web application server.

spring.session.timeoutHow long of inactivity the session will expiry. If a duration suffix is not specified, seconds will be used. 15m means 15 minutes. Default value is 1800. Tomcat only supports minute precision thus 170 will result in a 2 minutes timeout.
spring.session.jdbc.clean-cronSchedule for cleaning up expired session if use JDBC is used as store type. The value should be a cron expression.
spring.session.jdbc.same-datasource-with-rest-project

True - Use the data source defined for the API for session as well.

False - Use a data source other than the one defined for the REST project. spring.session.jdbc.datasource-identifier should be set when it is False.

It is only effective when store-type is JDBC.
spring.session.jdbc.datasource-identifierIdentifier for JDBC data source when it spring.session.jdbc.same-datasource-with-rest-project is FALSE.

Web Security Settings

PropertiesDescription
Enabled Security Feature

True - Enable security feature for this API.

False - Disable security feature for this API.
Security Authentication Sheet NameSecurity Authentication Sheet Name for this REST API. Two of this property can be added at most. One for user-password token. One for JWT.
Security User Details Service Sheet NameSecurity User Details Service Sheet Name for this REST API
Security End Point Url Authentication Sheet NameSecurity Resource Auth Sheet Name for this REST API

LDAP Settings

LDAP connection settings for this REST project. Multiple LDAP settings can be added supported. For example, spring.ldap.primary.url, primary is the identifier for this LDAP settings. To add another LDAP connection called test, add spring.ldap.test.url, spring.ldap.test.base, spring.ldap.test.admin-dn, and spring.ldap.test.admin-pwd.

PropertiesDescription
spring.ldap.primary.urlLDAP Server URL
spring.ldap.primary.baseLDAP Base information
spring.ldap.primary.admin-dnLDAP admin account. This is not needed in the current version. It is for future feature expansion when user details can be retrieved from LDAP server.
spring.ldap.primary.admin-pwdLDAP admin password. This is not needed in the current version. It is for future feature expansion when user details can be retrieved from LDAP server.

Microservice Configuration

Microservice is not a production-ready feature. Use it with caution.

PropertiesDescription
eureka.client.service-url.defaultZoneEureka server address

Maven Dependency

If your model use CallFunction to call other Java package, you should define the Java package you need in Maven Dependency section. When AlchemyJ generates the Java project, it will include the related Java library into the project. Each row in the table refers to one Java library.

ItemDescription
Maven Group IDThe group ID of the Java package.
Maven Artifact IDThe artifact ID of the Java package.
Maven VersionThe version number of the Java package your model needs
Scope

Compile - This should be the default scope. Compile dependencies are available in all class path of a project.

Provided - The dependency will be provided by the application server.

runtime - the dependency is needed for compilation, but is for execution.

For details, please refer to Maven official documentation.
