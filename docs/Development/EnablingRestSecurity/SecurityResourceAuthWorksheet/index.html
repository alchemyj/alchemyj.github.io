Security Resource Auth Worksheet

This worksheet defines the access control setting for endpoints.

Basic Settings

Configuration Type

Keep it as Rest Security End Point Url Authentication. Do NOT change this value.

End Point Url Authentication Type

  • Static - Endpoint access right configuration is set in this worksheet. Thus, it is fixed and cannot be changed after the API project has been built. Changing the access right configuration requires re-building the project to become effective.

  • Dynamic - Endpoint access right configuration will be loaded from an external source. Changing configuration can be done at the external source without rebuilding the project. In other words, the configuration is externalized.

Static End Point Url Authentication Configuration

Configure this section if you set the End Point Url Authentication Type to Static. You can configure one or more records here. The system will run through the rules from top to bottom. It will stop when the first match is found. For example,

/customer/tel
/customer/**

Suppose the configuration contains the above records. A request of http://customer/tel will match the first record since it is the first match.

URL Pattern

Define the URL pattern for configuration. Two asterisks can be used to indicate wildcard. For example,

/customer/**

It means all resources under [domain name]:[port]/customer/ would be matched.

Access Operator

The access type assigned.

  • Permit all - No access control. All requests to this endpoint will be allowed.

  • Authenticated - Only logged in users may gain access.

  • Has any authority - Only logged in users with any one of the authorities defined in the authority list may gain access.

  • Has both authority - Only logged in users with all authorities defined in the authority list may gain access.

Authority List

A list of authority values. Use comma (,) to separate multiple items.

Dynamic End Point Url Authentication Configuration

This table defines how endpoint access right configuration shall be loaded from an external source. Set the values in this table when End Point Url Authentication Type is Dynamic.

Loading Time

At which point shall Endpoint access right configuration be loaded.

  • On Server Start - Retrieves endpoint access right configuration upon application server startup.

  • By Request - Retrieves endpoint access right configuration on every request.

Loading Mode

How endpoint access right configuration will be loaded.

  • Database Standard Sql - Loads the configurations from a database by running a SQL statement.

  • Database Stored Procedure - Loads the configurations from a database by executing a stored procedure. Loading Time must be “By Request” as the stored procedure should return the authorization result to the request resource by the current user.

Is Use Same Data Source with Rest Project

  • True - Use the same data source of the REST project. You can find the configuration in Data Source Settings section of the Controller Worksheet. The Identifier name is primary.

  • False - Use another data source. You need to specify the Identifier name in Use Data Source Identifier.

Use Data Source Identifier

If Is Use Same Data Source with Rest Project is False, you need to configure another data source in Data Source Settings section of the Controller Worksheet. Then input the Identifier name here. Note that the Identifier name must be lowercase and cannot be primary, which has been used by the REST project already.

Loading SQL

The SQL statement for retrieving the dynamic endpoint access right configuration.

If Loading Time is “On Server Start”, the query should return following columns: “URL_PATTERN”, “ACCESS_OPERATOR” and “AUTHORITY_LIST”. No any parameter is provided for the query. It contains the same structure as defined in Static End Point Url Authentication Configuration.

If Loading Time is “By Request”, the query should return a single column “CHECK_RESULT”. The value 1 indicates the access of the requested resource is granted to the user. Other values indicate access denied. The following parameters may be used in the query.

  • REQUEST_URL - The parameter will be replaced by the request URL

  • AUTHENTICATED - The parameter will be replaced by TRUE or FALSE to indicate the user has logged in or not.

  • USERNAME - The parameter will be replaced by the user name of current user.

  • AUTHORITIES - The parameter will be replaced by the authorities that are granted to current user. Multiple value will be separated by comma (,). For example "Admin, Staff"

Loading Stored Procedure Name

The name of the stored procedure that should be called to return check result. Set this when Loading Mode is Database Stored Procedure. Such stored procedure should have the following input parameters:

  • REQUEST_URL - The parameter will be replaced by the request URL

  • AUTHENTICATED - The parameter will be replaced by TRUE or FALSE to indicate the user has logged in or not.

  • USERNAME - The parameter will be replaced by the user name of current user.

  • AUTHORITIES - The parameter will be replaced by the authorities that are granted to current user. Multiple values will be separated by comma (,). For example "Admin, Staff"

It should return an output parameter called “CHECK_RESULT”. The value 1 indicates the access of the requested resource is granted to the user. Other values indicate access denied.
