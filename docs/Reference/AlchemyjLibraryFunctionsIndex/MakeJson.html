MakeJson function

Description

The MakeJson function returns a JSON string given a corresponding name and value pair. If the value of the JSON name-value pair contains multiple records, a JSON array will be returned.

the following formula:

=MakeJson(“name”, “value”)

will formulate the JSON string and return the following:

{"name":"value"}

Syntax

MakeJson(Header, Data, [Data_type], [Exclude_column])

Argument Name Description

Header

(required)

The range of cells to be set as the name of the JSON name-value pair. It can be selected either vertically or horizontally.

You can also merge multiple columns into one by giving consecutive columns with the same name and index suffix as shown below

<field_name>__<running_number_from_01>:

For example, you have a field called Description, and it should be 60000 characters long which is longer than one cell in Excel can hold. Hence, two columns are, Description__01 and Description__02 are used. MakeJson will concatenate the values and the node name in the return JSON will be Description.

If a header name is surrounded by [], the content will be treated as the input parameter of a nested MakeJson call which returns an array of JSON object. For example, header is [phone] and data is A1:B1, A2:B4. This means A1 to B1 is the header range of the nested MakeJson call, and A2 to B4 are the three rows of data.

If a header name is surrounded by {}, the content will be treated as the input parameter of a nested MakeJson call which returns a JSON object.

{} and [] headers can create JSON strings with multiple level nodes. As the content expects a string which contains cell range address, it is recommended to use GetAddress to return an address range string. Refer to GetAddress for more details.

Data

(required)

The range of cells to be set as the value of the JSON name-value pair. It can be selected either vertically or horizontally but must be same as ‘Header’.

For example, if ‘Header’ is selected vertically, ‘Data’ must select vertically as well.

If a single record is selected, the result will be a JSON object.

If multiple records are selected, the result will be a JSON array.

Data_type

(Optional)

To specify the data type of the fields (‘Header’) in the format below. Following data type supported are in AlchemyJ, number, integer, boolean and string. Fields that do not exist will be ignored. Fields specified here will also be applied to nested MakeJson call using [] and {} headers.

<field_name>::<data_type>##<field_name>::<data_type>##...

Where,

## denotes AlchemyJ parameter separator.

:: denotes AlchemyJ parameter type separator.

For example, ID::number##Name::string

Please take note that the both separator symbols can be configured in %%SysConfig worksheet.

If you do not specify anything, default data type will follow the cell data type.

Exclude_column

(Optional)

To specify the fields to be excluded from the result. Fields that do not exist will be ignored. Fields specified here will also be applied to nested MakeJson call using [] and {} headers.

<field_name>##<field_name>##...

Where,

## denotes AlchemyJ parameter separator.

For example, ID#Last Name

Please take note that the parameter separator symbols can be configured in %%SysConfig worksheet.

If you do not specify anything, no fields will be excluded.

The function will return:

1) Content type: JSON object / JSON array

2) Method: within a cell / row cell array
