MakeJsonFromSchema function

Description

The MakeJsonFromSchema function returns a JSON string based on table relationship definition. If there are multiple tables with dependency, the JSON would be multi-level.

Syntax

MakeJsonFromSchema(Tables_relationship_header, Tables_relationship)

Argument NameDescription
Tables_relationship_header
(required)		To specify the range where defines the header of the relationship table.
Tables_relationship
(required)		To specify the range where defines the content of the relationship table. The empty row would be ignored.

Relationship Table

Column NameDescription
JSON Node Name
(optional)		1. This column is used to define the node name in the returned JSON.
2. If this column is not included or value is not inputted, generate the node name automatically using table name with following rule.
a. Change the first character of 'Table' column value to lower case.
b. Append "List" to the end if Multiple Row is TRUE.
c. for example, if the 'Table' column value is 'Client' and 'Multiple Row' is TRUE, will generate the node name as 'clientList'.
Table
(required)		1. Table name, It is mandatory column and the value is mandatory.
2. There is no validation about the Table name, which must be valid table name or a valid Excel Defined Name. We shall use Table Address to identify the table.
Table Address
(required)		1. The column is mandatory
2. The value is range address or Excel Defined Name of the table, header row should be included.
3. The referring table should include two rows at least, one for header and one data row. If the table address is invalid, will return #VALUE!
4. To enable high performance mode, it must to use GetAddress function to get the table address.
5. The blank
Parent Table
(optional)		1. Parent table name which is used to define the table dependency relationship
2. Parent table name should exist in 'Table' column value
3. If parent table name doesn't exist in 'Table' column value, return #VALUE!
4. Parent Table name could not be the same with 'Table' in the same row.
Key
(optional)		1. If Parent Table is filled in and parent table records is more than 1 row, Key is mandatory. If the parent table only has one record, Key can be omitted because it means all records in sub table will be joined.
2. The Key is a column name in the parent table, the column name should be same as data name defined in data dictionary.
3. If the Key is not provided when parent table is filled in, return #VALUE!
4. If the Key value cannot be matched with any column name in the parent table, the column and data of sub-table would not appear in the returned JSON.
Multiple Row
(optional)		1. Boolean value - True / False, which is to define the records in the table is single or multiple. And it can be inputted by user or calculate by the program
2. If the value is provided by user, will follow the inputted value
3. If this column is excluded or value is not inputted, check the no. of body rows of the table
a. Set to "True" if there is more than one body rows in the Table associated range.
b. Set to "False" if there is only one body row in the Table associated range.

4. If the inputted value is not a boolean value True / False, will return #VALUE!
Data Type Info
(optional)		1. The format is [Field Name] [Parameter Type Separator] [Data Type] [Parameter Separator]. E.g., Area Code::number##Tel::string
2. Data Type Info can be
a. number
b. integer
c. boolean
d. string

3. For date / datetime, it would be treated as a number (e.g. 31389 and 32176.47222) in returned JSON. You can convert it to text before making JSON. Or convert it back to date after received the JSON. Please note that the number of datetime can be converted to datetime properly in most of programming languages, but the number of date could only be recognized inside Excel. For example, Text( 31389, "yyyy-mm-dd")
4. Return #VALUE! if the string is in an unrecognizable format. E.g., Area Code::Date
5. The field name that does not exist would be ignored.
6. For more details about the data type conversion, please refer to the Data type conversion section below.

Please take note that the both separator symbols can be configured in %%SysConfig Worksheet.
Exclude Columns
(optional)		1. Define which columns in the table will not be included in the returned JSON.
2. The format is [Field Name] [Parameter Separator]. E.g. Area Code##Tel
3. The field name that does not exist will be ignored.
Data Dictionary Name
(optional)		1. The address of the Data Dictionary where defines the formating rules. For example, DataDictionary!A11:Y21. Please include the header in the range of cells in the first row. When a data dictionary component is added to a workbook, it automatically create a name, AlchemyJ_DataDic that specifies the data item list area. Thus, you can simply use [DataDictionarySheetName]!AlchemyJ_DataDic here. For example, if the data dictionary worksheet name is CustomerDataDic, the name here would be CustomerDataDic!AlchemyJ_DataDic.
2. It is used to format the value in the JSON only. Will not trigger data dictionary validation
3. When the Data Dictionary is provided, all columns should be included in the Data Dictionary. Otherwise, it will be omitted from the returned JSON.
4. If both "Data Type Info" and "Data Dictionary Name" are provided, the logic of "Data Type Info" would be followed.
5. If Data Type in "Data Type Info" is not provided, but Data Dictionary Name provide the date type, the data value cell type would be used for converting.
6. To enable the high performance mode, it must use GetAddress function to define the data dictionary.

Data type conversion

During the processing of making JSON. Some data will be converted from a type to another. Following are the rules of the conversion.

Convert From TypeConvert To TypeConversion Rules
stringbooleanconvert to boolean if value = true/false, otherwise, #VALUE!
stringintegerconvert to integer if it is numeric, otherwise #VALUE!
stringnumberconvert to number if it is numeric, otherwise, #VALUE!
For example "010" to 10
booleanstringconvert to "true" or "false"
booleanintegerconvert true to 1 and false to 0
booleannumberconvert true to 1 and false to 0
numberstringconvert to the string representation of the number with trailing zeros. For example 2.3 to "2.3".
numberbooleanconvert 0 to false, otherwise, true
numberintegerround the number. (1.4 to 1, 1.5 to 2)

If the cell value is Excel error (e.g. #VALUE!, #NA!, #DIV/0), the corresponding node would be omitted.

The function will return:

1) Content type: JSON array

2) Method: within a cell / row cell array. It split the returned JSON string into multiple cells, when the string length is longer than a single cell character limit - 32767, the remaining string will be extended in the row array.
