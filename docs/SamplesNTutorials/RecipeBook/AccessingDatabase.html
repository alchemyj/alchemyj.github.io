Accessing database

You may need database access if your API needs to persist large amounts of data or query data directly from a database. AlchemyJ lets you execute SQL statements and stored procedures using ExecuteSql and ExecuteSP. Currently, it supports Oracle and MySQL. This recipe demonstrates the basic of using ExecuteSql to retrieve and insert customer data.

Setting up database access in the workbook

Before we start, let us define the database connection we will be using in this workbook. Go to System Configuration worksheet (%%SysConfig) and set up the database connection to be used in AlchemyJ Studio. The two key we need to set are DB Connection String and Database Type.

Put the database connection string of your database in DB Connection String. For Database Type, you may put either MySQL or Oracle depending on the database you are going to you.

Case 1 - Retrieving a single record

This example uses table "tb_customer" in database "aj_sample_mysql57". The following records exist in the table.

Create an AlchemyJ workbook similar to the one below. It has a worksheet named GetCustomer and a function point called Get Customer.

With all that set up, let's add logic to retrieve customer data from table tb_customer. ExecuteSQL executes a SQL statement against a database. So we shall compose the following SQL statement and put it in cell B10. It retrieves customer, Merry So, from the table. This cell will be referred by ExecuteSQL in later steps.

SELECT CUSTOMER_NAME, DOB, GENDER, CUSTOMER_LVL, TELEPHONE FROM TB_CUSTOMER WHERE CUSTOMER_NAME ='MERRY SO';

We would like the retrieved records to be placed in cell B4 to F4. Select cells B4 to F4 and input the following formula (same as the one as shown in B7) and press CTRL+ SHIFT+ ENTER. This will make the formula becomes an Array Formula so that the data to be retrieved can be placed into the selected cell range.

=IF(AlchemyJ_FunctionPoint = "Get Customer", ExecuteSql(B10), "")

We use the above formula to execute the SQL statement we just inputted in B10. Please note that formula may be triggered automatically when refreshing depending on your formula calculation setting. This is not desirable as you might not want the SQL statement to be executed whenever you modify the formula or hit the calculate button in Excel. That is why we use an IF function to make sure ExecuteSQL will only run when AlchemyJ_FunctionPoint = "Get Customer" which is the time we click Run Function Point and select the Get Customer function point.

We are now ready to test our work. Run function point Get Customer and see the result.

You should see the retrieved records in B4:F4.

Case 2 - Retrieving multiple records

In this example, we will create a function that retrieves multiple records from the same table. Start by creating a new worksheet named GetCustomers, and adding a new function point called Get Customers in class definition.

Put the following SQL statement in B15 which will be used by ExecuteSql.

SELECT CUSTOMER_NAME, DOB, GENDER, CUSTOMER_LVL, TELEPHONE FROM TB_CUSTOMER;

As our table only has 5 records, select B4:F9 and input the following formula as shown in B12. Press CTRL+ SHIFT+ ENTER to make it becomes an Array Formula. If your table contains more than 5 records, make sure you select a range large enough for the returned rows.

=IF(AlchemyJ_FunctionPoint = "Get Customers", ExecuteSql(B15), "")

To test the function, run function point Get Customers. You should see the results in B4:F9.

Case 3 - Creating a record

So far you have seen how to retrieve data from a database using ExecuteSQL. In this example, you will use ExecuteSQL to insert a record into the customer table. Start by creating a new worksheet named CreateCustomer, and adding a new function point called Create Customer in class definition.

To keep this example simple, we will use a static SQL statement which contains the customer information we want to insert. In reality, the data to be inserted often comes from another cells, such as input parameters of your function. In such case, you should use the parameter arguement in ExecuteSQL to define the data instead of using a dynamic SQL statement as this can prevent SQL Injection attacks.

To create a record with data as shown in B4:F4, key in the following SQL statement in B10 which will be used by ExecuteSql.

INSERT INTO TB_CUSTOMER (CUSTOMER_NAME, DOB, GENDER, CUSTOMER_LVL, TELEPHONE) VALUES ('MICKY MAI','1988-05-08','F','3','22206666');

Select C12 and input the following formula just as shown in B7.

=IF(AlchemyJ_FunctionPoint = "Create Customer", ExecuteSql(B10), "")

Now we are all set. Run the function point Create Customer to see the result.

The value returned in C12 by ExecuteSQL is the number of affected records. 1 means one record has been inserted. It indicates the operation was successful.

At last, you can run the Get Customers function and verify that the record has been created into to database.

Check out the AlchemyJ Excel Extended Function index if you want to learn more about the advance usage of ExecuteSql and ExecuteSP.
