Raising exceptions

Good APIs should properly report errors when problem occurs. There are many scenarios that error might occur. Wrong configuration setting, failed connection to a 3rd party API, and invalid input are a few examples. There are two types of errors in AlchemyJ. They are System error and Model error.

Getting a list of system errors

System errors are errors raised by AlchemyJ Excel Extended Functions such as ExecuteSql, AuthenLdap, and CallWsRest. When an error occurs in an AlchemyJ Excel Extended Function, it creates an error record on the System Error List. You may use GetSystemError to retrieve and search for errors on the System Error List.

Take AuthenLdap as an example, it will raise an error if server name is an invalid LDAP server name. You can use =GetSystemError() to retrieve the error list.

GetSystemError([cell_reference], [function_name], [error_message])

In this example, we leave all the parameters blank so that it retrieves all errors on the System Error List. GetSystemError returns a list of records with 3 columns. Thus, it should be entered as an array formula by selecting a 3-column range, type in the formula, and hit CTRL+SHIFT+ENTER.

Raising and getting a list of model errors

Beside system errors, there may be times you want raise an error manually. For example, when you catch a bad input or override a system error.

In this example, we use AuthenLdap to validate a user name and password against a LDAP server. AuthenLdap will return FALSE if the username and password do not match. We want to return a specific error message when this happens. This can be achieved by using RaiseError.

RaiseError(error_message, [error_code], [cell_reference], [run_by_function_point_only])

In this example, we use "Invalid User Name or Password" as the error message, "100001" as the error code, and GetAddress(C8) as the reference address.

After the error is raised by RaiseError, it can be retrieved by using GetModelError.

GetModelError(cell_reference,error_code,error_message)

In this example, we specify error code as "100001" so that only errors with error code "100001" will be returned. GetModelError returns a list of records with 3 columns. Thus, it should be entered as an array formula by selecting a 3-column range, type in the formula, and hit CTRL+SHIFT+ENTER.

Retrieving errors returned from an AlchemyJ REST API

AlchemyJ returns status code 502 when any error exists on the System Error List and Model Error List. The error content will be shown under the messages node.

Retrieving errors returned from an AlchemyJ Java method

try{
      String telephoneNum = biz.queryUserTelephone();
   }catch(AlchemyjFormulaExecutionException ajEx) {
      // Write your logic to handle the exception above
   }catch(Exception e){
      // Handle other exception
   }

Configuring REST API error handling behaviour

The following properties in Rest Controller worksheet can control how a REST API behaves when errors occur:

  • alchemyj.stopOnFormulaError
  • alchemyj.stopOnUserRaiseError
  • alchemyj.throwOnFormulaError
  • alchemyj.throwOnUserRaiseError

Refer to Controller Worksheet for details.
