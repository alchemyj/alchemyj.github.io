Reading application configuration

There are many good reasons to externalize your API configuration. For example, AuthenLDAP requires you to set the server name and domain. These values should not be hard-coded in your Excel model. The API would be more robust if these values can be loaded from an external file which administrators may modify when the API is deployed in different environments.

Setting it up in Excel

  1. Go to the Application Configuration worksheet (%%AppConfig) and define the name of the property you want to add in the Name column.

  2. The value column is where the property value will be imported from an external source. If a value is set here, it becomes the default value.

  3. The value can be referenced in a business logic worksheet.

Modifying configuration values for REST APIs

AlchemyJ REST API can load application configuration from application.yml. Follow the below steps to create and modify the yml file.

  1. If you have not created the application.yml file for your API before, compile the REST API first.

  2. In the output folder, locate \src\main\resources\application.yml. Make a copy of this file and put it in the folder as specified in the application configuration file location in REST Package Definition.

  3. Open this application.yml file with a text editor or an yml editor of your choice.

  4. You can find the application properties under alchemyj\applicationProps. Modify their values as needed.

  5. When the REST API is loaded, the values you just changed in application.yml will be loaded into the %%AppConfig worksheet.

  6. AlchemyJ look for the application.yml file in the following orders:/#basic-settings

    • YML file as specified by Application Configuration File Environment Variable in REST Controller worksheet
    • YML file as specified by Application Configuration File Location in REST Controller worksheet
    • If none of the above exists, it will load the values from the application.yml inside the jar file.

Setting configuration values for Java APIs

Java classes do not have an application configuration file as in REST API. Application configuration is usually loaded from your application via yml file, xml file, or from a database instead. When initializing an AlchemyJ class in Java, use the following code to load the configuration values.

  private Map<String, String> createApplicationProperties() {
        Map<String, String> result = new LinkedHashMap<>();
        result.put("Property_name_1", "192.168.5.149");
        result.put("Property_name_2", "25.0");
        result.put("Property_name_3", "sample@gmail.com");
        return result;
      }

  public void init(){
        AlchemyjClassContextHolder.getInstance().getClassInitPropertyHolder().setApplicationPropertiesMap(createApplicationProperties());
      
      }
