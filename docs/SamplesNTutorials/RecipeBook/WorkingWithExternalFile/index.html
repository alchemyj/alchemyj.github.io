Reading and writing text files

There are times your API needs to read and write files to disk. For example, you may need import data from a flat file generated by another application or generate an interface file for a downstream system. In AlchemyJ, ReadFile and WriteFile can do just that.

Reading data from a file

In this example, you will learn how to load data from a CSV to a business logic worksheet. The below screen shot shows the CSV file that will be loaded.

  1. Prepare an input file similar to the one below. Save it at a location such as D:/AlchemyJ/Test-case/TestData/ReadFile/Customer.csv (or .txt. The file extension does not matter).

  1. ReadFile has seven parameters. In this example, we only need to define the first two parameters of ReadFile. In C3, input the text file location. In C4, put a comma (,) since we want ReadFile to separate values into columns by comma.

     ReadFile(file_location, column_separator, [left_enclosure_character], [right_enclosure_character], [begin_at_line], [number_of_lines], [character_set], [run_by_function_point_only])

  2. As ReadFile returns an array of records, highlight the range you want the data to be returned, type in =ReadFile(C3,C4), and press CTRL+SHIFT+ENTER to enter it as an array formula. The data will be imported as below when you run any function point.

If you want ReadFile to run whenever Excel calculates the workbook, set the last parameter of ReadFlie, runningFlagControl, to FALSE.

Writing data to a file

Now let’s try to export some data from an Excel worksheet a file. This is very easy to do using WriteFile.

  1. Prepare the date you want to write. In this example, we will export data from cell A27 to B40.

  2. WriteFile can supports many options

    WriteFile(data, file_location, [write_option], [column_separator], [left_enclosure_character], [right_enclosure_character], [character_set], [number_of_retry], [interval_of_retry], [run_by_function_point_only])

    In this example, we only need to define

    • Data: B10:C23
    • File_location: A full file path as defined in C4.
    • Write_option: Set it to 1 to overwrite the existing file content if any.
    • Column_separator: Put comma here as we want WriteFile to separate column values by a comma. WriteFile will return TRUE if it writes the file successfully and will return FALSE otherwise. It is a single value return so it does not need to be entered as an array formula like ReadFile does.

  1. The data will be saved to the specified path when you run any function point.
